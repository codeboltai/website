## Common Questions

### How is CodeBolt different from GitHub Copilot?

**Copilot** is a single-agent coding assistant that provides sequential suggestions, delivering ~1.2x productivity gains.

**CodeBolt** is a multi-agent swarm platform where 10+ specialized agents work in parallel, delivering 5-10x productivity gains through coordinated teamwork.

[Detailed comparison](./comparison.md#github-copilot)

### How is CodeBolt different from Cursor?

**Cursor** is an AI-first code editor with deep code understanding and refactoring capabilities, but limited to single-agent workflows (~1.5x productivity).

**CodeBolt** is a complete development environment with multi-agent swarms coordinating through stigmergy, enabling parallel development (5-10x productivity).

[Detailed comparison](./comparison.md#cursor)

### Can I really get 5-10x productivity gains?

Yes. Real-world success scenarios demonstrate:
- Startup: MVP in 2 weeks (vs 4 weeks planned)
- Enterprise: 8-month modernization (vs 18 months planned)
- Emergency: 16-minute MTTR (vs 2-hour industry average)

These gains come from **parallel development** - 10 agents working simultaneously vs 1 developer working sequentially.

[See real-world examples](./success-scenarios.md)

### What's the stigmergy coordination mechanism?

Stigmergy is nature-inspired coordination (from ant colonies) where agents communicate indirectly through "pheromones" - digital signals left on tasks, files, and code elements. This enables:

- No central orchestrator (no single point of failure)
- Linear scaling (O(n) vs O(n²) for direct messaging)
- Emergent intelligence (complex coordination from simple rules)
- Graceful degradation (agent failures only affect local work)

[Deep dive into stigmergy](./key-features.md#2-stigmergy-based-agent-coordination)

### How does the infinite context work?

CodeBolt overcomes LLM context limitations through six specialized memory types:
1. **Semantic Memory**: Vector-based code understanding across entire codebases
2. **Episodic Memory**: History of changes, decisions, and outcomes
3. **Working Memory**: Current task context and immediate goals
4. **Procedural Memory**: Established patterns and best practices
5. **Social Memory**: Agent interactions and collaboration history
6. **Encyclopedia Memory**: Domain knowledge and documentation

This enables agents to access unlimited context despite fixed LLM context windows.

[Learn about memory systems](./key-features.md#4-infinite-context-through-six-memory-types)
