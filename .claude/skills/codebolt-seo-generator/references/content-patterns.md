# SEO Content Patterns

Proven patterns for writing high-converting SEO content.

```
"CodeBolt vs [Competitor]: [Unique Differentiator]"

```

```
"How [Persona] Can [Benefit] with CodeBolt"
"Why [Persona] Love CodeBolt"
```

### Tech Stack Pages
```

"[Stack] Development: [Benefit] with CodeBolt"
```

### Industry Pages
```
"CodeBolt for [Industry]: [Primary Benefit]"

"Digital Transformation in [Industry] with AI"
```

## Content Frameworks

### AIDA (Attention-Interest-Desire-Action)
```markdown
## [Attention-grabbing headline]

[Hook that captures attention]

### [Interest-building subheading]
[Build interest with problems/challenges]

### [Desire-creating subheading]
[Create desire with benefits/results]

### [Action-oriented subheading]
[Clear call to action]
```

### PAS (Problem-Agitate-Solution)
```markdown
## [Problem Statement]

[Describe the problem clearly]

### Why This Matters
[Agitate the problem with details]
[List consequences]
[Quantify the cost]

### The Solution
[Present CodeBolt as solution]
[Explain how it works]
[Show results]
```

### Before-After-Bridge
```markdown
## Before CodeBolt

[Describe current state]
[List challenges]
[Show limitations]

## After CodeBolt

[Describe future state]
[List benefits]
[Show possibilities]

## How to Get There

[Present CodeBolt as bridge]
[Explain the process]
[Call to action]
```

## Keyword Integration Patterns

### Primary Keyword Placement
- Title tag (first 50 characters)
- H1 heading
- First paragraph (within 100 words)
- One or more H2 headings
- Naturally throughout content (1-2% density)
- URL slug
- Meta description
- Alt text of images

### Secondary Keyword Placement
- H2/H3 headings
- Body paragraphs
- Image alt text
- Internal link anchor text

### Long-Tail Keyword Placement
- FAQ questions
- Subheadings
- Natural sentences
- Image captions

## Content Length Guidelines

| Page Type | Minimum Words | Optimal Words | Notes |
|-----------|---------------|---------------|-------|
| Comparison | 800 | 1,200-1,500 | Need detailed comparison |
| Use Case | 600 | 1,000-1,200 | Focus on problem/solution |
| Persona | 600 | 1,000-1,200 | Personalize for audience |
| Tech Stack | 500 | 800-1,000 | Technical depth |
| Industry | 700 | 1,000-1,200 | Industry-specific context |

## Writing Style Guidelines

### Tone
- Professional but approachable
- Confident without arrogance
- Authoritative but friendly
- Technical but accessible

### Voice
- Active voice preferred
- Direct addressing ("you")
- Clear, simple language
- Avoid jargon unless defined

### Structure
- Short paragraphs (2-3 sentences)
- Mixed sentence length
- Bullet points for lists
- Subheads every 200-300 words

### Do's
- ✅ Use concrete numbers and metrics
- ✅ Include specific examples
- ✅ Add relevant testimonials
- ✅ Link to supporting pages
- ✅ Optimize for skimmability

### Don'ts
- ❌ Make vague claims
- ❌ Overuse superlatives
- ❌ Keyword stuff
- ❌ Write wall of text
- ❌ Duplicate other pages

## Heading Hierarchy

```markdown
# H1: Primary keyword + benefit (only one per page)

## H2: Main sections (3-5 per page)

### H3: Sub-sections (as needed)

#### H4: Rare, only for very long content
```

```markdown

[Explanation]
```

### The "Vs" Formula
```markdown

| Aspect | CodeBolt | [Competitor] |
|--------|----------|-------------|
| [Dimension] | [CodeBolt advantage] | [Competitor limitation] |
```
