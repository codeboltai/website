Architecture

Codebolt is built from the ground up. Here's how.

The 5-Layer Stack

Layer 1 — Editor Interface surface
Layer 2 — Agent Runtime runtime
Layer 3 — Drift Engine Rust
Layer 4 — Narrative Versioning git-tree
Layer 5 — Governance & Coordination Loop core

Most competitors stop at Layer 2.

Technical Deep-Dives

Traditional multi-agent systems rely on direct message passing — agents send instructions to each other, creating fragile coupling and scaling bottlenecks. Codebolt uses stigmergy: agents communicate indirectly through structured shared state. Each agent reads from and writes to a shared environment, allowing coordination to emerge without explicit messaging. This is how biological swarms scale to millions of agents — and it's how Codebolt scales beyond the limits of message-passing architectures.

The narrative engine is a Rust-based system that maintains a complete record of agent actions, decisions, and reasoning. It uses git trees for snapshot versioning, enabling agent attribution (which agent changed what and why), cross-environment coordination (reconciling work across branches and contexts), and full replay capability. Every change carries its narrative — the intent behind it, not just the diff.

Drift tracking operates at two levels. The local Rust engine instances assess drift within individual agent contexts — structural drift (architecture diverging from plan), scope drift (features expanding beyond intent), constraint drift (violations of specified rules), and coverage drift (gaps in implementation). The global JavaScript coordination layer aggregates these signals across all agents, detecting systemic drift patterns that no single agent would catch alone.

SCAAB (Structured Context for Agent-Addressable Bundles) packages encode domain knowledge, architectural decisions, and operational constraints into portable bundles that agents can consume. They provide structured context without requiring agents to process an entire codebase. Bundles are composable, versionable, and designed for cross-agent knowledge sharing.

IntentGraph captures what should happen — a directed graph of goals, constraints, and success criteria. PlanGraph captures how it should happen — a dependency-ordered execution plan derived from the IntentGraph. Together, they provide the reference frame for drift detection: as agents execute, the system continuously compares actual outcomes against the IntentGraph to catch deviation early.
