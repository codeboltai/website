Core Concept

Four Ways to
Coordinate Agents

Different coordination approaches for different needs. Choose what fits your workflow—or combine them.

⚡

Parallel Agents

Multiple independent agents working on unrelated tasks simultaneously. No coordination overhead—just pure parallelism.

Best for:

Independent features, batch processing, isolated components

Coordination: None (tasks are independent)
🌳

Sub-Agents

Parent agent spawns child agents for subtasks. Hierarchical decomposition with clear ownership.

Best for:

Complex tasks that need decomposition, recursive problems

Coordination: Parent-child hierarchy
🎯

Orchestrator Mode

Central orchestrator agent that assigns tasks and monitors progress. Classic hub-and-spoke coordination.

Best for:

Small teams (≤50 agents), tight coordination requirements

Coordination: Central control
🐜

Stigmergy Swarms

Decentralized coordination through environmental signals. No central controller. Agents self-organize through pheromones.

Best for:

Large swarms (200+ agents), complex emergent behavior

Coordination: Environmental signals (stigmergy)

Hybrid

Combine Modes

Modes aren't mutually exclusive. Use orchestrator for a small coordinated team, while stigmergy coordinates across teams.

Example: Frontend team (5 agents + orchestrator) and Backend team (8 agents + orchestrator) coordinate through stigmergic pheromones at the API boundary.
