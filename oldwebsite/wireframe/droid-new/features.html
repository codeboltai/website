Autonomous Coordination

Self-organizing intelligence through stigmergy, deliberations, and smart job management.

Stigmergy-Based Coordination

Nature-inspired coordination without central orchestrators. Agents communicate through pheromone signals:

request_split (Purple)

Signals that a task should be decomposed into smaller sub-tasks

importance (Amber)

Indicates task priority and urgency levels

saturation (Red)

Shows when a task is being worked on by multiple agents

takeup_interest (Emerald)

Expresses agent interest in taking up a task

task_not_ready (Orange)

Marks tasks with blocking dependencies

job_available (Green)

Signals that blockers have been resolved

files_blocked (Red)

Indicates file conflicts or reservation intents

workingonit (Blue)

Shows active work in progress

Agent Deliberation System

Structured discussion mechanisms for collective decision-making:

  • Voting Deliberations: Democratic decision making on architectural choices
  • Feedback Deliberations: Collect opinions on proposals and implementations
  • Q&A Sessions: Knowledge sharing and problem-solving discussions
  • Shared Lists: Collaborative list building for test cases, requirements, etc.

Job Coordination System

Intelligent task management preventing conflicts and enabling fair distribution:

  • Job Locking: Prevents race conditions when agents start work
  • Job Bidding: Agents bid on tasks they're well-suited for
  • Job Blockers: Track and resolve dependencies automatically
  • Split Proposals: Agents propose decomposing complex tasks
  • Unlock Requests: Polite mechanism for requesting lock release

Perfect Memory

Infinite context through IDE-level memory management. AI that never forgets.

Memory Types

Multiple specialized memory systems for different use cases:

  • Episodic Memory: Event-based memory recording what happened, when, and by whom
  • Semantic Memory: Structured knowledge in the Knowledge Graph about concepts and relationships
  • JSON Memory: Structured data storage with schema support
  • Markdown Memory: Long-form text storage for documentation and notes
  • Context Memory: Thread-specific working memory for active conversations
  • Todo Memory: Task tracking within memory with completion status

Knowledge Graph (Kuzu-based)

Graph database understanding of your entire codebase:

  • Instance Templates: Schema definitions for records and edges
  • Record Kinds: Entity types (STRING, INT64, DOUBLE, BOOL, DATE, TIMESTAMP, UUID, etc.)
  • Edge Types: Relationship definitions between entities
  • Query Views: Cypher-like queries for pattern matching and traversal
  • System-Level Understanding: Agents see relationships, not just isolated files

Vector Database

Semantic search across all project knowledge:

  • Document Embeddings: Store documents with vector representations
  • Hybrid Search: Combine semantic similarity with keyword matching
  • Metadata Filtering: Filter searches by tags, types, or custom metadata
  • Similarity Search: Find related code by meaning, not just keywords

Cross-Generational Learning

Breakthrough externalized memory architecture:

  • 87% Knowledge Survival: Knowledge retained across 50+ agent generations
  • 38% Improvement: Problem-solving improves over time through cumulative learning
  • 67% Faster Adaptation: Agents adapt to new situations more quickly
  • Zero Internal State: Agents maintain no memory—everything externalized for collective intelligence
