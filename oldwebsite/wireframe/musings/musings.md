Application Commandments (how to think about the application): Agent Behaviour
- It is not the agents that have to be smart, it is the environment that shapes the agents
     (प्रकाशक्रियास्थितिशीलं भूतेंद्रियात्मकं भोगापवर्गार्थं दृश्यम्॥) - "The seen (prakṛti) has the nature of illumination (sattva), activity (rajas), and inertia (tamas); it consists of the elements and sense organs; its purpose is experience (bhoga) and liberation (apavarga) for the Purusha (the seer)."
- The agent should not be managing too much life and context the environment should be
- Agent can have conflicts the democratic institutions should be able to resolve them
- User is not a God, the user should work with the agent colonies
- Agent is a malicious entity, the environment should be able to protect itself using guardrails.
- User shoiuild be god in terms of visibility, but with 100s of agents running the user should get new ways to understand what is going on.
- The agents should not be burdened with understanding the whole application. They should do their tasks and the environment will take of it.
        (कर्मण्येवाधिकारस्ते मा फलेषु कदाचन, मा कर्मफलहेतुर्भूर्मा ते सङ्गोऽस्त्वकर्मणि॥)
