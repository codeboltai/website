Stigmergy-Based Coordination

Self-organizing intelligence inspired by nature. Agents coordinate through environmental signals—no central orchestrator required.

What is Stigmergy? Stigmergy is indirect coordination through environmental signals—the same mechanism that allows ant colonies to build complex structures without a central commander. In Codebolt, agents leave "pheromone" signals that other agents can sense and respond to, creating emergent organization without any coordination overhead.

Real-World Analogy

Like Ant Colonies. Like Markets.

Stigmergy is all around us. Understanding these real-world examples helps explain how Codebolt agents coordinate.

🐜 Ant Colonies

Ants leave pheromone trails to food sources. Other ants follow strong trails and reinforce them. No ant directs others—collective behavior emerges from simple signals.

📈 Market Economies

Prices signal scarcity and demand. Producers and consumers respond to prices without central direction. Market outcomes emerge from individual decisions based on signals.

🚗 Traffic Flow

Drivers see brake lights (signals) and slow down. This propagates backward, creating traffic waves. No central control—emergent coordination through signals.

Pheromone System

Agents deposit pheromones to signal state, priority, and intent. Other agents sense these signals and act accordingly. Each pheromone type serves a specific coordination purpose.

importance
Signal job priority
Decay: None (sticky)
High priority tasks attract more agents
request_split
Signal job should be split
Decay: None (sticky)
Agents see complex tasks needing decomposition
saturation
Signal job is heavily worked on
Decay: Slow
Prevents too many agents on same task
takeup_interest
Signal interest in job
Decay: Moderate
Agents signal they're considering a task
task_not_ready
Signal blocking dependencies
Decay: None (sticky)
Agents avoid tasks with blockers
job_available
Signal blockers resolved
Decay: Slow
Attracts agents to newly available tasks
files_blocked
Signal files reserved
Decay: None (sticky)
Prevents file-level conflicts
completion
Signal task completion
Decay: Fast
Signals downstream tasks can start
Pheromone Dynamics

How Pheromone Decay Works

Not all signals should last forever. Pheromone decay controls how long signals persist, enabling temporal coordination.

No Decay (Sticky)

Important signals that must persist: importance, blockers, file locks. These remain until explicitly cleared.

Slow Decay

Signals that fade over hours: saturation, job availability. Allows gradual redistribution of agents.

Fast Decay

Transient signals: completion, interest. Fades in minutes, enabling rapid re-coordination.

Smart Job Coordination

Intelligent task management where agents claim work, signal intentions, and avoid conflicts automatically.

🔒 Job Locking

When an agent starts work, it locks the job—preventing others from duplicating effort. Other agents can request unlock if they have higher priority work.

🗳️ Job Bidding

Agents can bid on tasks they are suited for. Bids include reasoning, priority score, and current status. Best bidder wins—no race conditions.

🔄 Split Proposals

Complex tasks can be split. Agents propose splits with reasoning. Multiple proposals compete. Best proposal wins via deliberation.

💬 Blocker Management

Track dependencies between jobs. Mark blockers with pheromones. Auto-signal when blockers resolve. Downstream tasks auto-start.
