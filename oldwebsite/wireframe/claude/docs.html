Architecture Overview

Codebolt is built as a distributed system designed from the ground up for multi-agent software development. Understanding the architecture helps you deploy, scale, and integrate Codebolt effectively.

System Components

46 Specialized Panels
From SwarmPanel to CodePanel to PreviewPanel—each UI component provides specific visibility and control over different aspects of agent operations.
93 Backend Services
Microservices architecture handling agent lifecycle, memory management, coordination, and execution. Each service is independently scalable.
Memory Layer
Kuzu graph database for relationships, custom vector DB for semantic search, SQLite for structured data, and KV store for caching.
Coordination Layer
Pheromone-based stigmergy system, deliberation engine, job market, and file locking—enabling zero-overhead agent coordination.

Technology Stack

  • Frontend: React + TypeScript, Zustand state management, Monaco Editor
  • Backend: Node.js with 93 specialized microservices
  • Databases: Kuzu (graph), Custom Vector DB, SQLite, KV Store
  • Communication: WebSocket for real-time, MCP protocol support
  • Desktop: Electron application with native OS integration
API Reference

Codebolt provides comprehensive REST APIs for managing agents, jobs, pheromones, memory, and deliberations. All APIs are authenticated and support both direct calls and webhooks.

Agent Management API

GET /api/v1/agents

List all agents with their status, capabilities, and current assignments.

POST /api/v1/agents

Create a new agent with specified configuration, permissions, and model selection.

PATCH /api/v1/agents/:id

Update agent configuration, pause/resume, or redirect to new task.

Pheromone API

POST /api/v1/pheromones

Deposit a pheromone signal on a job or file. Types: importance, saturation, task_not_ready, job_available, etc.

GET /api/v1/pheromones/:jobId

Retrieve all active pheromone signals for a specific job.

Memory System API

POST /api/v1/memory/episodic

Store episodic memory events with timestamps, agent IDs, and context.

GET /api/v1/memory/semantic

Query semantic memory using vector similarity search.

GET /api/v1/memory/graph

Query the knowledge graph for entity relationships and dependencies.

Deliberation API

POST /api/v1/deliberations

Create a deliberation: voting, feedback, Q&A, or shared list.

POST /api/v1/deliberations/:id/vote

Submit a vote or feedback on an active deliberation.

Agent SDK

Create custom agents programmatically using our TypeScript/JavaScript SDK. Define agent behaviors, capabilities, and integration points with full type safety.

Installation

npm install @codebolt/agent-sdk # or yarn add @codebolt/agent-sdk

Creating a Custom Agent

import { Agent, Tool } from '@codebolt/agent-sdk'; // Define custom tools const dependencyCheck = new Tool({ name: 'dependency_check', description: 'Check for vulnerable dependencies', execute: async (context) => { // Implementation } }); // Create agent const securityAgent = new Agent({ name: 'Security-Agent', model: 'claude-3-opus', capabilities: ['security_audit', 'vulnerability_scan'], tools: [dependencyCheck], systemPrompt: `You are a security specialist agent. Your role is to identify security vulnerabilities, check dependencies, and ensure best practices.`, permissions: { read: ['src/**', 'package.json'], write: [], } }); // Register agent await securityAgent.register();

Agent Lifecycle Hooks

  • onStart: Called when agent begins a task
  • onProgress: Called with progress updates
  • onComplete: Called when task completes
  • onError: Called on task failure
  • onDeliberation: Called when agent is asked to deliberate

Multi-Model Configuration

Agents can use different models for different tasks:

const agent = new Agent({ models: { reasoning: 'claude-3-opus', coding: 'gpt-4-turbo', fast: 'claude-3-haiku' }, modelSelector: (task) => { if (task.requiresComplexReasoning) return 'reasoning'; if (task.isCoding) return 'coding'; return 'fast'; } });
Integration Guide

Codebolt integrates seamlessly with your existing development tools and workflows. Configure webhooks, CI/CD pipelines, and communication platforms.

CI/CD Integration

Configure Codebolt to participate in your CI/CD pipeline:

  • Agents run tests before deployment
  • Security agents scan on PR creation
  • Documentation agents auto-update on merge
  • Review agents provide automated code review
# GitHub Actions example name: Codebolt Agent Review on: pull_request: types: [opened, synchronize] jobs: agent-review: runs-on: ubuntu-latest steps: - name: Trigger Codebolt Agents run: | curl -X POST https://api.codebolt.io/webhooks/ci \ -H "Authorization: Bearer ${{ secrets.CODEBOLT_TOKEN }}" \ -d '{"event": "pr_review", "pr_id": "${{ github.event.pull_request.number }}"}'

Deployment Guide

Deploy Codebolt on your infrastructure or use our managed cloud. Self-hosting gives you complete data control and customization.

Self-Hosting Requirements

Minimum
• 4 CPU cores
• 16GB RAM
• 100GB SSD
• Docker/Kubernetes
Recommended (100+ agents)
• 32 CPU cores
• 128GB RAM
• 1TB NVMe SSD
• Kubernetes cluster

Quick Start (Docker)

git clone https://github.com/codebolt/codebolt.git cd codebolt docker-compose up -d # Access at http://localhost:3000

Environment Configuration

# Core settings CODEBOLT_HOST=0.0.0.0 CODEBOLT_PORT=3000 # Database GRAPH_DB_PATH=/data/kuzu VECTOR_DB_PATH=/data/vector SQLITE_DB_PATH=/data/codebolt.db # LLM Providers (multi-model support) OPENAI_API_KEY=sk-... ANTHROPIC_API_KEY=sk-ant-... # Security JWT_SECRET=your-secret-key ENCRYPTION_KEY=your-encryption-key # Scaling MAX_AGENTS=100 CONCURRENT_JOBS=50

Scaling Considerations

  • Each agent requires ~500MB RAM base + model context
  • Vector DB performance scales with available RAM
  • Graph DB benefits from SSD storage
  • Horizontal scaling: multiple worker nodes
  • Vertical scaling: more agents per node
Performance Tuning

Optimize Codebolt for your specific use case. Balance agent count, memory configuration, and response time.

Agent Count Optimization

  • Small projects (<10k LOC): 10-20 agents optimal
  • Medium projects (10k-100k LOC): 20-50 agents
  • Large projects (>100k LOC): 50-100+ agents

Memory Configuration

  • Episodic memory: Configure retention period (default: forever)
  • Vector cache: Allocate RAM for semantic search
  • Graph DB: Tune relationship indexing
  • KV Store: Configure TTL for ephemeral data

Pheromone Tuning

  • Decay rates: Adjust how fast signals fade
  • Thresholds: Set minimum signal strength
  • Weights: Prioritize certain pheromone types

Monitoring

  • Real-time metrics dashboard
  • Agent performance tracking
  • Memory usage monitoring
  • Pheromone signal visualization
